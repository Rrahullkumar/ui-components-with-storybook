# InputField Component

A comprehensive, accessible input component built with React, TypeScript, and Tailwind CSS.

## Overview

The InputField component provides a consistent, flexible input solution with built-in validation, multiple visual variants, and extensive customization options.

## Best Practices

### Accessibility
- Always provide meaningful labels
- Use helperText for additional context
- Provide clear error messages
- Ensure sufficient color contrast

### Performance
- Use controlled components sparingly for large forms
- Debounce validation for better UX
- Consider lazy loading for complex forms

### Design
- Choose variants consistently across your app
- Use appropriate sizes for context
- Maintain consistent spacing

## Props Documentation

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `label` | `string` | - | Input label text |
| `placeholder` | `string` | - | Placeholder text |
| `helperText` | `string` | - | Additional help text |
| `errorMessage` | `string` | - | Error message (overrides helperText) |
| `variant` | `'outlined' \| 'filled' \| 'ghost'` | `'outlined'` | Visual style variant |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | Input size |
| `disabled` | `boolean` | `false` | Disable input |
| `invalid` | `boolean` | `false` | Show invalid state |
| `loading` | `boolean` | `false` | Show loading state |
| `clearable` | `boolean` | `true` | Show clear button |
| `type` | `string` | `'text'` | HTML input type |
| `maxLength` | `number` | - | Maximum character limit |
